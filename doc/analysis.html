3 Accessing Undirected Graphs

3.1 Iterator<Vertex> adjacentVertices (Vertex v)

Since the main loop executes V times and there are two Θ(1) checks per run, the runtime is Θ(2V) The storage space is O(V * V) for an extremely well connected graph.

3.2 Iterator<Edge> incidentEdges (Vertex v)

Since the main loop exceutes V times and there are two Θ(1) checks, as well as constant assignments and constructions the runtime of the algorithm is Θ(2V).

The storage space is O(V * V) for a well-connected graph.

5 Mutators

5.1 addVertex()

Since the matrix is initialized with a certain size, adding a vertex would entail expanding the matrix. This means cloning the matrix, making a new matrix with one more row/column, and copying over the contents, the last function having a runtime of Θ(V2)

5.2 deleteVertex()

Deleting the vertex entails shifting the row elements over and subtracting the number of vertices the matrix has. If the first vertex is deleted, one would have to shift V-1 rows and V-1 columns, which would mean that the runtime would be Θ(V2)

5.3 insertEdge() / insertDirectedEdge()

Since inserting something into a HashMap is on average O(1), the runtime is Θ(1). The storage space required is two integers and a new edge/pair object

5.4 setDirectionFrom() / setDirectionTo()

Accessing the endpoints of an edge is constant, as well as setting the values in the matrix, so the runtime of the two methods are Θ(1)

5.5 reverseDirection()

Since getting and setting values in the HashMap and Matrix is constant the runtime is Θ(1)
